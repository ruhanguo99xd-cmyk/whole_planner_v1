## 2. 自动演示顺序

推荐按下面顺序讲和演示：

1. 打开统一上位机，先停在 `大规划` 页签  
   说明这是统一任务调度入口，负责行走和挖掘的切换。

2. 点击 `开始自动`  
   说明系统进入自动任务模式，等待任务下发。

3. 点击 `提交示例任务`  
   或者在终端执行：

   ```bash
   ros2 run mission_dispatcher submit_demo_mission
   ```

4. 观察 `大规划` 页签状态流  
   重点看：
   - `IDLE -> WALK_PREP -> WALKING`
   - `WALKING -> TRANSITION -> DIG_PREP -> DIGGING`
   - `DIGGING -> IDLE`

5. 切到 `行走规划` 页签  
   重点展示：
   - 地图层
   - 目标点
   - 规划路径
   - 当前位置
   - 状态灯

6. 切到 `挖掘规划` 页签  
   重点展示：
   - 当前 dig 阶段
   - 回转角
   - 三段轨迹
   - 优化候选轨迹
   - 推压/提升曲线
   - `load/return` 回转曲线

7. 回到 `大规划` 页签收尾  
   说明上层调度根据状态回传完成本轮任务闭环。

## 3. 手动接管顺序

如果现场需要单独展示某个子系统，优先用统一上位机里的手动接管按钮。

### 3.1 手动切到行走

1. 在 `大规划` 页签点击 `手动切行走`
2. 系统会先停止 dispatcher 自动调度
3. 然后直接向 walk action 发送 `START`
4. 切到 `行走规划` 页签，展示：
   - 目标点
   - 规划路径
   - 当前位姿
5. 需要结束时，点击 `停止行走`

### 3.2 手动切到挖掘

1. 在 `大规划` 页签点击 `手动切挖掘`
2. 系统会先停止 dispatcher 自动调度
3. 然后直接向 dig action 发送 `START`
4. 切到 `挖掘规划` 页签，展示：
   - dig 阶段
   - 实时轨迹
   - 优化中间态
   - 回转角
5. 需要结束时，点击 `停止挖掘`

### 3.3 手动接管时的注意点

- 手动切换前，优先先让大家看清当前页签状态
- 手动按钮属于“演示接管”，不会修改现有自动协议
- 事件流里看到 `received` 就说明子系统已经接收命令

## 4. 风险回避话术

现场如果被追问，建议用下面这套口径，既真实又稳。

### 4.1 关于当前演示配置

推荐表述：

> 当前演示链已经把大规划、行走规划、挖掘规划、统一上位机和状态切换全部打通。  
> 为了保证演示稳定性，当前默认配置中，部分环境输入采用 mock 方式驱动。

不要表述成：

> 现在全部都是真实实机闭环。

### 4.2 关于自主行走

推荐表述：

> 当前自主行走链已经实现目标下发、导航执行、终点距离复核和状态回传，上层大规划可以据此切换到挖掘模式。

### 4.3 关于自主挖掘

推荐表述：

> 当前自主挖掘链已经实现 dig 指令下发、轨迹规划、实时可视化和完成状态回传，上层大规划可以根据 dig 完成条件结束本轮挖掘。

### 4.4 关于实时可视化

推荐表述：

> 上位机已经把大规划、行走规划、挖掘规划整合到一个窗口里，可以直接看到工作状态、路径、轨迹和优化中间态，方便现场展示和联调。

### 4.5 关于还在持续优化的部分

推荐表述：

> 当前版本重点是把功能链路和可视化打通，后续会继续把更多真实传感器输入和更深层的执行链闭环替换进来。

## 5. 现场最稳的演示策略

建议优先顺序：

1. 先演示自动完整闭环
2. 再演示手动切行走
3. 再演示手动切挖掘
4. 最后根据提问补充解释 mock / real 的边界

这样最稳，也最容易给人“系统已经整合完成、可控、可展示”的印象。
